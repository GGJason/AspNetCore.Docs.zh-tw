## <a name="how-does-it-work"></a><span data-ttu-id="2df60-159">它的運作狀況如何？</span><span class="sxs-lookup"><span data-stu-id="2df60-159">How does it work?</span></span>

<span data-ttu-id="2df60-160">瀏覽器連結會使用 SignalR 建立 Visual Studio 和瀏覽器之間的通訊通道。</span><span class="sxs-lookup"><span data-stu-id="2df60-160">Browser Link uses SignalR to create a communication channel between Visual Studio and the browser.</span></span> <span data-ttu-id="2df60-161">啟用瀏覽器連結時，Visual Studio 就會當做多個用戶端 （瀏覽器使用） 可以連接到的 SignalR 伺服器。</span><span class="sxs-lookup"><span data-stu-id="2df60-161">When Browser Link is enabled, Visual Studio acts as a SignalR server that multiple clients (browsers) can connect to.</span></span> <span data-ttu-id="2df60-162">瀏覽器連結也會註冊 ASP.NET 要求管線中的中介軟體元件。</span><span class="sxs-lookup"><span data-stu-id="2df60-162">Browser Link also registers a middleware component in the ASP.NET request pipeline.</span></span> <span data-ttu-id="2df60-163">此元件會插入特殊`<script>`到每個頁面要求來自伺服器的參考。</span><span class="sxs-lookup"><span data-stu-id="2df60-163">This component injects special `<script>` references into every page request from the server.</span></span> <span data-ttu-id="2df60-164">您可以看到選取的指令碼參考**檢視原始檔**瀏覽器和向下捲動到結尾`<body>`標記的內容：</span><span class="sxs-lookup"><span data-stu-id="2df60-164">You can see the script references by selecting **View source** in the browser and scrolling to the end of the `<body>` tag content:</span></span>

        {"requestId":"a717d5a07c1741949a7cefd6fa2bad08","requestMappingFromServer":false}

<span data-ttu-id="2df60-165">不修改原始程式檔。</span><span class="sxs-lookup"><span data-stu-id="2df60-165">Your source files aren't modified.</span></span> <span data-ttu-id="2df60-166">中介軟體元件會以動態方式插入指令碼參考。</span><span class="sxs-lookup"><span data-stu-id="2df60-166">The middleware component injects the script references dynamically.</span></span> 

<span data-ttu-id="2df60-167">因為瀏覽器端程式碼是所有 JavaScript，所以它適用於所有瀏覽器 SignalR 支援而不需要瀏覽器外掛程式。</span><span class="sxs-lookup"><span data-stu-id="2df60-167">Because the browser-side code is all JavaScript, it works on all browsers that SignalR supports without requiring a browser plug-in.</span></span>
