<span data-ttu-id="e0711-140">gRPC 訊息會使用[Protobuf](https://developers.google.com/protocol-buffers/docs/overview)序列化，這是一種有效率的二進位訊息格式。</span><span class="sxs-lookup"><span data-stu-id="e0711-140">gRPC messages are serialized using [Protobuf](https://developers.google.com/protocol-buffers/docs/overview), an efficient binary message format.</span></span> <span data-ttu-id="e0711-141">Protobuf 會非常快速地在伺服器和用戶端上序列化。</span><span class="sxs-lookup"><span data-stu-id="e0711-141">Protobuf serializes very quickly on the server and client.</span></span> <span data-ttu-id="e0711-142">在小型訊息承載中 Protobuf 序列化結果，這在有限的頻寬案例（例如行動應用程式）中很重要。</span><span class="sxs-lookup"><span data-stu-id="e0711-142">Protobuf serialization results in small message payloads, important in limited bandwidth scenarios like mobile apps.</span></span>
