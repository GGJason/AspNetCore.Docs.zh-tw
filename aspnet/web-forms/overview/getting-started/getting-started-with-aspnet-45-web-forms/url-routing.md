### <a name="url-patterns"></a><span data-ttu-id="80d94-131">URL 模式</span><span class="sxs-lookup"><span data-stu-id="80d94-131">URL Patterns</span></span>

<span data-ttu-id="80d94-132">URL 模式可以包含常值和變數的預留位置 （稱為 URL 參數）。</span><span class="sxs-lookup"><span data-stu-id="80d94-132">A URL pattern can contain literal values and variable placeholders (referred to as URL parameters).</span></span> <span data-ttu-id="80d94-133">常值和預留位置位於其以斜線分隔 URL 區段 (`/`) 字元。</span><span class="sxs-lookup"><span data-stu-id="80d94-133">The literals and placeholders are located in segments of the URL which are delimited by the slash (`/`) character.</span></span>

<span data-ttu-id="80d94-134">您 web 應用程式進行要求時，URL 會剖析為區段和預留位置，和變數的值提供給要求處理常式。</span><span class="sxs-lookup"><span data-stu-id="80d94-134">When a request to your web application is made, the URL is parsed into segments and placeholders, and the variable values are provided to the request handler.</span></span> <span data-ttu-id="80d94-135">此程序是在剖析查詢字串中的資料和傳遞給要求處理常式的方式類似。</span><span class="sxs-lookup"><span data-stu-id="80d94-135">This process is similar to the way the data in a query string is parsed and passed to the request handler.</span></span> <span data-ttu-id="80d94-136">在這兩種情況下，為 URL 中包含變數的資訊，並將其傳遞給索引鍵-值組表單中的處理常式中。</span><span class="sxs-lookup"><span data-stu-id="80d94-136">In both cases, variable information is included in the URL and passed to the handler in the form of key-value pairs.</span></span> <span data-ttu-id="80d94-137">查詢字串索引鍵和值是在 URL 中。</span><span class="sxs-lookup"><span data-stu-id="80d94-137">For query strings, both the keys and the values are in the URL.</span></span> <span data-ttu-id="80d94-138">路由的索引鍵中的 URL 模式中，定義的預留位置名稱，而只有值在 URL 中。</span><span class="sxs-lookup"><span data-stu-id="80d94-138">For routes, the keys are the placeholder names defined in the URL pattern, and only the values are in the URL.</span></span>
