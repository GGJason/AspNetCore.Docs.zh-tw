## <a name="conventions"></a><span data-ttu-id="6991d-329">慣例</span><span class="sxs-lookup"><span data-stu-id="6991d-329">Conventions</span></span>

<span data-ttu-id="6991d-330">您必須撰寫 Entity framework 能夠為您建立完整的資料庫中的程式碼數量非常小，因為*慣例*，可讓 Entity Framework 的假設。</span><span class="sxs-lookup"><span data-stu-id="6991d-330">The amount of code you had to write in order for Entity Framework to be able to create a complete database for you is minimal because of *conventions*, or assumptions that Entity Framework makes.</span></span> <span data-ttu-id="6991d-331">其中一些已註明，或用來而您知道它們的存在：</span><span class="sxs-lookup"><span data-stu-id="6991d-331">Some of them have already been noted or were used without your being aware of them:</span></span>

- <span data-ttu-id="6991d-332">複數化的形式的實體類別名稱會用於資料表名稱。</span><span class="sxs-lookup"><span data-stu-id="6991d-332">The pluralized forms of entity class names are used as table names.</span></span>
- <span data-ttu-id="6991d-333">實體屬性名稱會用於資料行名稱。</span><span class="sxs-lookup"><span data-stu-id="6991d-333">Entity property names are used for column names.</span></span>
- <span data-ttu-id="6991d-334">名為實體屬性`ID`或是*classname* `ID`被視為主索引鍵屬性。</span><span class="sxs-lookup"><span data-stu-id="6991d-334">Entity properties that are named `ID` or *classname* `ID` are recognized as primary key properties.</span></span>
- <span data-ttu-id="6991d-335">是否將它命名，將會解譯為外部索引鍵屬性的屬性*&lt;導覽屬性名稱&gt;&lt;主索引鍵屬性名稱&gt;* (比方說，`StudentID`如`Student`導覽屬性，因為`Student`實體的主索引鍵是`ID`)。</span><span class="sxs-lookup"><span data-stu-id="6991d-335">A property is interpreted as a foreign key property if it's named *&lt;navigation property name&gt;&lt;primary key property name&gt;* (for example, `StudentID` for the `Student` navigation property since the `Student` entity's primary key is `ID`).</span></span> <span data-ttu-id="6991d-336">外部索引鍵屬性也可以命名相同只要&lt;主索引鍵屬性名稱&gt;(例如`EnrollmentID`因為`Enrollment`實體的主索引鍵是`EnrollmentID`)。</span><span class="sxs-lookup"><span data-stu-id="6991d-336">Foreign key properties can also be named the same simply &lt;primary key property name&gt; (for example, `EnrollmentID` since the `Enrollment` entity's primary key is `EnrollmentID`).</span></span>

<span data-ttu-id="6991d-337">您已了解可以覆寫慣例。</span><span class="sxs-lookup"><span data-stu-id="6991d-337">You've seen that conventions can be overridden.</span></span> <span data-ttu-id="6991d-338">例如，指定資料表名稱不應該要複數化，，和更新版本，您會看到如何明確地標示為外部索引鍵屬性的屬性。</span><span class="sxs-lookup"><span data-stu-id="6991d-338">For example, you specified that table names shouldn't be pluralized, and you'll see later how to explicitly mark a property as a foreign key property.</span></span> <span data-ttu-id="6991d-339">您將進一步了解慣例和如何覆寫其[建立更多複雜的資料模型](creating-a-more-complex-data-model-for-an-asp-net-mvc-application.md)稍後在本系列教學課程。</span><span class="sxs-lookup"><span data-stu-id="6991d-339">You'll learn more about conventions and how to override them in the [Creating a More Complex Data Model](creating-a-more-complex-data-model-for-an-asp-net-mvc-application.md) tutorial later in this series.</span></span> <span data-ttu-id="6991d-340">如需慣例的詳細資訊，請參閱 <<c0> [ 程式碼的第一個慣例](/ef/ef6/modeling/code-first/conventions/built-in)。</span><span class="sxs-lookup"><span data-stu-id="6991d-340">For more information about conventions, see [Code First Conventions](/ef/ef6/modeling/code-first/conventions/built-in).</span></span>
